---
id: TIME_TRAVEL
title: "Time travel."
essence: "Append corrections, never erase; any past state can be reconstructed by replaying events."
---

All state is rewindable via events. Think accounting and ledgers. No mutations!

Accountants never erase. They append corrections. If every state change is an immutable event, you can reconstruct any past state by replaying events up to that point. Debug by rewinding. Audit by querying history. Recover by replaying.
