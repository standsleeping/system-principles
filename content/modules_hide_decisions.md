---
id: MODULES_HIDE_DECISIONS
title: "A module is a hidden decision."
essence: "A module hides a decision behind a stable interface, so the implementation can change freely without disturbing its users."
---

A module isn't a subroutine. It's a decision, an assumption, or a source of change, hidden behind an interface which protects the module's users from that source of change.

Beneath every interface is a river of iteration: new information, better approaches, lessons learned. The interface provides cover, a stable surface that lets the implementation churn without disturbing its users. The interface changes slowly; the code behind it changes freely.
