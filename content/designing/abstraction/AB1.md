True abstraction is about separating concerns that don't belong together,
not about hiding complexity behind a facade.

## Rationale

When concerns are intertwined, changes to one affect the other unexpectedly.
Good abstraction untangles these concerns so each can evolve independently.

## Application

1. When complexity feels overwhelming, look for hidden entanglement
2. Ask: "Are multiple concerns mixed together here?"
3. Separate the concerns first, then consider if hiding is needed
